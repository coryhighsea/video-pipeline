# Video Shorts Pipeline

An internal tool that turns Google Meet recordings into edited, captioned short-form videos for LinkedIn and YouTube Shorts. Drop in a recording, review AI-suggested clips, and export polished mp4s ‚Äî automatically transcribed, LLM-edited, and captioned.

---

## How It Works

### Step 1 ‚Äî Upload

Open `http://localhost:3030` and upload:

1. **Optional context** ‚Äî type any guidance for Grok in the textarea before uploading (e.g. "focus on the incident reporting section", "this is a long-form YouTube video, not a standup meeting")
2. **Select the transcript** ‚Äî click "Pick .txt" and choose the Gemini-generated notes file (auto-generated by Google Meet after the call)
3. **Drop the video** ‚Äî drag the raw `.mp4` recording onto the upload zone, or click to select it

Once uploaded, the server parses the Gemini `.txt` format ‚Äî timestamps appear on their own line (`00:01:05`), followed by speaker lines (`Cory Hisey: ...`). These are converted into a prompt-ready string with millisecond markers (`[1:05 = 65000ms]`) and analysis starts immediately.

---

### Step 2 ‚Äî Analyze

All heavy processing runs automatically. Two Grok calls happen per job, plus Whisper and ffmpeg per clip.

#### 2a. Grok 1 ‚Äî Clip suggestions

Grok (`grok-4-1-fast-non-reasoning`) reads the formatted transcript and returns a structured JSON object validated against a Zod schema. For each clip it returns:

- **`title`** ‚Äî punchy, under 60 characters, suitable for LinkedIn/YouTube
- **`rationale`** ‚Äî 2‚Äì3 sentences on why the moment works standalone
- **`segments[]`** ‚Äî array of `{ startMs, endMs }` time ranges (1‚Äì4 per clip). Multiple segments let you stitch a topic mentioned early with its conclusion later, or skip dead air between two strong moments.
- **`totalDurationSeconds`** ‚Äî sum of all segment durations; Grok is instructed to keep this between 30‚Äì90s

If `customContext` was provided at upload, it is appended to the prompt as "Additional context from the editor".

**Note:** Gemini timestamps are only accurate to ¬±30 seconds. They're sufficient for Grok to identify the right moments; Whisper handles precise word-level timing next.

#### 2b. Per-clip Whisper transcription

For each suggested clip, rather than transcribing the full recording, only the audio windows belonging to the clip are extracted:

```bash
ffmpeg -i video.mp4 -ss {startSec} -t {durationSec} -ar 16000 -ac 1 -c:a pcm_s16le segment.wav
```

Each segment window is transcribed independently by `whisper.cpp` with `tokenLevelTimestamps: true`, giving word-level start/end times. Timestamps are offset back to the original video timeline by adding `seg.startMs` to every word. All segment results are merged into a single captions array.

#### 2c. Grok 2 ‚Äî LLM-driven edit decisions

The word-level captions are sent to Grok (`grok-4-1-fast-reasoning`) along with the clip's title and rationale. The word list includes pause durations between consecutive words and trailing silence at segment ends:

```
Segment 1 [5000ms ‚Äì 35000ms]:
  [5234-5456] "we"
  [5456-5678] "need"
  <pause 450ms>
  [6128-6340] "um"
  [6340-6550] "to"
  ...
  <trailing silence 800ms>
```

Grok returns `{ startMs, endMs }[]` intervals to remove. Because it understands context, it can distinguish:
- "like" as a verbal tic (cut) vs. "like" in a comparison (keep)
- "so" as a filler pause (cut) vs. "so" as a logical connector (keep)
- Meaningful pauses before key points (keep) vs. dead air (cut)
- False starts and self-corrections (cut)

The returned intervals are validated, merged (overlapping within 50ms combined), and stored in `clips.removedIntervals`.

#### 2d. ffmpeg cut

The Grok 2 removal intervals are used to build keep windows ‚Äî the portions of each segment to retain. These are assembled into an ffmpeg `filter_complex` command:

```
[0:v]trim=start=S:end=E,setpts=PTS-STARTPTS[v0];
[0:a]atrim=start=S:end=E,asetpts=PTS-STARTPTS[a0];
...
[v0][a0][v1][a1]...concat=n=N:v=1:a=1[outv][outa]
```

Output: `public/gap-edited-{id}.mp4` ‚Äî a single continuous clip starting at PTS 0.

#### 2e. Caption remapping

After cutting, every removed interval shifts subsequent word timestamps earlier. The remapper accounts for three offset sources:

1. **Pre-segment offset** ‚Äî a clip starting at 5:00 has Whisper timestamps at ~300000ms, but the output starts at PTS 0
2. **Inter-segment gaps** ‚Äî content between segments is not in the output
3. **Grok 2 removals** ‚Äî filler/silence cuts shift remaining words earlier

```
remappedMs = originalMs - cumulativeRemovedBefore(originalMs)
```

Words that fall entirely within any removed interval are dropped. Result saved to `public/captions-{id}-remapped.json`.

By the time the job reaches `stage1_review`, all clips are fully processed and ready to render.

---

### Step 3 ‚Äî Review

Each clip appears as a card showing:
- **Editable title**
- **Segment timestamps** ‚Äî adjust M:SS start/end values, add or remove segments
- **Grok's rationale** ‚Äî visible at all stages including after rendering (useful for writing upload titles and descriptions)

Actions per clip:
- **Approve** ‚Äî mark ready to render
- **Reject / Undo** ‚Äî hide from list
- **Re-edit** (on rendered clips) ‚Äî reset back to suggested, adjust segments, render again as `slug-v2.mp4`
- **Dismiss ‚úï** (on rendered clips) ‚Äî hide the card from the UI; rendered file is kept on disk

Job-level actions:
- **Context field + Re-analyze** ‚Äî update the Grok context and re-run the full analyze pipeline. New clip suggestions are added alongside existing ones.
- **Delete job** ‚Äî removes DB record and all files from disk (upload, transcript, public copy, gap-edited mp4s, captions, rendered outputs). Blocked if any clip is currently rendering.

---

### Step 4 ‚Äî Render

Click **"Render N Clips ‚Üí"**. All editing is already done ‚Äî this step is just Remotion.

For each approved clip, a full-quality 1080√ó1920 Remotion render runs using the `PipelineMultiClip` composition:

```bash
bunx remotion render PipelineMultiClip out/YYYY-MM-DD/{slug}.mp4 \
  --props '{"videoSrc":"gap-edited-{id}.mp4","captionsFile":"captions-{id}-remapped.json","segments":[{"startMs":0,"endMs":N}]}' \
  --concurrency 4
```

The composition overlays `CaptionOverlay` ‚Äî a React component that loads the remapped captions JSON, groups words into TikTok-style pages (~1.4s each), and renders word-highlight subtitles 240px from the bottom (above platform UI chrome). Captions are **baked into the mp4**.

Render progress is throttled to 10% milestones in the server terminal and browser log.

Output: `out/YYYY-MM-DD/{slug}.mp4`. Re-renders of the same clip increment to `slug-v2.mp4`, `slug-v3.mp4`, etc., keeping previous versions.

## Setup

### Requirements

- [Bun](https://bun.sh) runtime
- PostgreSQL
- ffmpeg (must be in PATH)
- whisper.cpp ‚Äî downloaded automatically on first analyze run

### First-time setup

```bash
# 1. Create the database
createdb videos_pipeline

# 2. Install server dependencies
cd server
bun install

# 3. Create environment file
cat > .env << EOF
PIPELINE_DATABASE_URL=postgres://localhost/videos_pipeline
XAI_API_KEY=your_xai_key_here
EOF

# 4. Run migrations
bunx drizzle-kit generate
bunx drizzle-kit migrate

# 5. Start the server
bun run server
```

Open `http://localhost:3030`.

## Output Files

| Path | Description |
|------|-------------|
| `uploads/{uuid}.mp4` | Original uploaded video |
| `uploads/{uuid}-transcript.txt` | Original Gemini transcript |
| `public/{uuid}.mp4` | Video copy served to Remotion via `staticFile()` |
| `public/gap-edited-{id}.mp4` | ffmpeg output ‚Äî Grok 2 edits applied, starts at PTS 0 |
| `public/captions-{id}-remapped.json` | Word captions remapped to gap-edited video timeline |
| `out/YYYY-MM-DD/{slug}.mp4` | Final full-quality 1080√ó1920 export |
| `out/YYYY-MM-DD/{slug}-v2.mp4` | Re-rendered version (original kept) |

## Tech Stack

| Layer | What |
|-------|------|
| Server | Hono on Bun, port 3030 |
| Database | PostgreSQL via Drizzle ORM |
| AI ‚Äî clip selection | xAI Grok 1 (`grok-4-1-fast-non-reasoning`) via Vercel AI SDK + Zod schema validation |
| AI ‚Äî edit decisions | xAI Grok 2 (`grok-4-1-fast-reasoning`) ‚Äî context-aware filler/silence removal |
| Transcript parsing | Custom parser for Gemini `.txt` format |
| Transcription | whisper.cpp via `@remotion/install-whisper-cpp`, per-segment windowing |
| Video editing | ffmpeg `filter_complex` with `trim`/`atrim` + `concat`, driven by Grok 2 intervals |
| Caption rendering | `@remotion/captions` `createTikTokStyleCaptions`, baked into mp4 |
| Video rendering | Remotion `PipelineMultiClip`, `--concurrency 4` |
| UI | Vanilla HTML/JS, no build step |
